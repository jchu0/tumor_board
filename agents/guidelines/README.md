# Guidelines Agent

The **guideline-coverage sub-check** of the tumor-board assistant. Internally
it is a small agent (deterministic triage → specialist lookups over an on-disk
EBM "shelf" → one Claude synthesis call). Externally it is **one function**:

```python
from agents.guidelines import check_guideline_coverage

findings = check_guideline_coverage(patient)   # -> list[dict]  (contract A)
```

It takes a patient file, decides which guideline domains apply, consults
specialist checks that look up the shelf, and returns a list of **findings** —
each stating what needs to be addressed, with a **two-axis, reasoned
confidence**.

> **Placeholder content.** The shelf cards under `shelf/` and the patient under
> `fixtures/placeholder_patient.json` are minimal, clearly-labeled
> **PLACEHOLDERS** so the pipeline runs end-to-end and the tests pass. The
> physician on the team replaces them with real EBM content and a real synthetic
> case. This repo half owns only the *machinery*, not the medicine.

---

## The confidence model (the core idea)

Two **separate, orthogonal** signals — deliberately never collapsed into one
number:

| Field | What it means | Where it comes from |
|---|---|---|
| `recommendation_grade` | Strength of the **evidence itself** (Class of Recommendation / Level of Evidence, e.g. `"I / A"`). | **Straight from the shelf card, byte-for-byte.** Never invented, inferred, or modified by the model. |
| `match_confidence` + `confidence_rationale` | The agent's **own certainty that this guideline applies to THIS patient** (right biomarker, stage, complete data). | **Generated by the one Claude call**, reasoning over the patient chart. The rationale is required and must cite specific chart facts. |

> "The evidence is strong (I / A), but I'm only *moderately* confident this is
> the right biomarker for this patient." — the model supplies the second half;
> the shelf supplies the first. The language model **never** states a
> recommendation or grade from its own memory.

## Contract (A) — Findings output (boundary with the orchestrator)

`check_guideline_coverage` returns `list[dict]`. Every finding has **exactly**
these ten string fields:

| Field | Meaning |
|---|---|
| `domain` | e.g. `"fertility_preservation"` |
| `issue` | what needs to be addressed / what's missing |
| `recommendation` | the recommendation (verbatim from the card) |
| `evidence_ref` | which guideline card backs this (verbatim from the card) |
| `recommendation_grade` | `"I / A"` — Class of Recommendation / Level of Evidence (verbatim from the card) |
| `match_confidence` | `"high"` \| `"moderate"` \| `"low"` |
| `confidence_rationale` | WHY that confidence — cites specific chart facts (required, non-empty) |
| `patient_facing_note` | plain language, includes numbers where present |
| `status` | `"gap"` \| `"addressed"` \| `"uncertain"` |
| `source_agent` | e.g. `"guidelines_agent/fertility"` |

`contract.py::validate_finding` is the machine-checked source of truth for this
shape; every finding is validated before it leaves `check_guideline_coverage`.

Example finding:

```json
{
  "domain": "fertility_preservation",
  "issue": "Fertility preservation may apply but is not documented.",
  "recommendation": "[PLACEHOLDER] Offer fertility-preservation counseling / referral before starting gonadotoxic therapy.",
  "evidence_ref": "[factice] Placeholder fertility-preservation card — replace with a real guideline citation.",
  "recommendation_grade": "I / A",
  "match_confidence": "moderate",
  "confidence_rationale": "Chart shows cancer=breast, age=34, planned_therapy_class=gonadotoxic, satisfying the card's applies_when.",
  "patient_facing_note": "Because chemotherapy can affect fertility, we should discuss fertility-preservation options before treatment.",
  "status": "gap",
  "source_agent": "guidelines_agent/fertility"
}
```

## Contract (B) — EBM card schema (boundary with the physician)

Each card is an **"applies-when" half** plus a **"then-flag" half**. One JSON
file per specialist under `shelf/` (a file holds one card or a list). Every
`applies_when` key is optional — an absent key means "no constraint".

```jsonc
{
  "id": "fertility-PLACEHOLDER-001",       // unique id
  "specialist": "fertility_preservation",  // which specialist reads this card
  "applies_when": {                        // every key optional; absent = no constraint
    "cancer": ["breast"],                  // list: patient value must intersect
    "stage": ["I", "II", "III"],           // list
    "age_min": 18,                         // numeric lower bound (inclusive)
    "age_max": 45,                         // numeric upper bound (inclusive)
    "planned_therapy_class": ["gonadotoxic"]
  },
  "recommendation": "…",                   // copied verbatim into the finding
  "recommendation_grade": "I / A",         // copied verbatim — the model never touches it
  "evidence_ref": "[factice] …",           // copied verbatim

  // OPTIONAL fields:
  "requires": ["ER", "PR", "HER2"],        // completeness checklist (biomarker card)
  "staleness_days": 180,                   // goals-of-care staleness threshold
  "addressed_when": {                      // how the chart shows it's already handled
    "any_resource_text": ["fertility", "oncofertility"]
  },
  "domain": "…",                           // OPTIONAL override of the finding's domain
  "issue": "…"                             // OPTIONAL override of the finding's issue
}
```

### Field notes for authoring

- **`applies_when`** — list keys match case-insensitively and require the patient
  to actually have a matching value (a stated constraint the chart can't back up
  means the card does not fire). `age_min` / `age_max` are inclusive bounds.
- **`requires`** (biomarker-completeness) — the list of biomarkers that must be
  documented. Reference data lives here, never hard-coded in the code.
- **`staleness_days`** (goals-of-care) — how old a documented goals-of-care
  conversation may be before it counts as a gap.
- **`addressed_when.any_resource_text`** — keywords; if any appears anywhere in
  the patient file, the item is treated as already addressed (`status:
  "addressed"` instead of `"gap"`). Omit it and each specialist falls back to
  sensible built-in keywords — so you're never *forced* to write it.

### Specialists and their shelf files

| Specialist (`specialist`) | Kind | File | Uses |
|---|---|---|---|
| `guideline_coverage` | pure lookup | `shelf/guideline_coverage.json` | `applies_when`, `addressed_when` |
| `fertility_preservation` | pure lookup | `shelf/fertility_preservation.json` | `applies_when`, `addressed_when` |
| `germline_testing` | pure lookup | `shelf/germline_testing.json` | `applies_when`, `addressed_when` |
| `biomarker_completeness` | completeness | `shelf/biomarker_completeness.json` | `applies_when`, `requires` |
| `goals_of_care` | staleness | `shelf/goals_of_care.json` | `applies_when`, `staleness_days`, `addressed_when` |

## Patient file schema (input)

FHIR-*flavoured*: resources grouped by type, mirroring the Abridge
`synthetic-ambient-fhir-25` convention. For the demo matcher, each resource
carries a few plain convenience fields (`cancer`, `stage`, `biomarker`,
`therapy_class`) that a real FHIR mapping layer would derive from coded fields.

```jsonc
{
  "id": "PT-001",
  "patient_context": { "patient": { "gender": "female", "age": 34 } },
  "encounter_fhir": {
    "resources_by_type": {
      "Condition":        [ { "cancer": "breast", "stage": "II" } ],
      "Observation":      [ { "biomarker": "ER" }, { "biomarker": "PR" } ],
      "MedicationRequest":[ { "therapy_class": "gonadotoxic" } ],
      "CarePlan":         [ { "title": "goals of care discussion", "date": "2023-01-10" } ]
    }
  }
}
```

- Age comes from `patient_context.patient.age`, else computed from `birthDate`.
- A top-level `"features": { ... }` block overrides anything extraction found —
  the simplest hook for testing or for a caller that already has flat features.

## Architecture (v1 — reliability first)

```
patient (FHIR-ish JSON)
        │
        ▼
  extract_features            matcher.py   — FHIR-ish → flat matchable attrs
        │
        ▼
  select_specialists          triage.py    — deterministic, from patient fields
        │
        ▼
  specialists[*].run(patient) specialists/ — look up the shelf via loader+matcher,
        │                                    decide gap/addressed → PartialFinding
        ▼
  synthesize(patient, partials) synthesis.py — ONE Claude call adds match_confidence,
        │                                       confidence_rationale, patient_facing_note
        ▼
  validate_finding(*)         contract.py  — enforce contract (A)
        │
        ▼
  list[dict]  (contract A)  → orchestrator
```

- **Triage** is deterministic (no model). Each specialist owns a
  `triage_applies(features)` predicate.
- **Specialists** are clean, independently-callable `run(patient)` functions
  registered in `specialists/__init__.py`. A future Claude tool-use loop can
  replace the deterministic triage by exposing each `run` as a tool — **without
  rewriting the specialists**.
- **One** Claude call does synthesis. It only ever fills the three judgment
  fields; `recommendation` / `recommendation_grade` / `evidence_ref` are copied
  verbatim from the card.

## Running it

Requires **Python 3.11+** (developed on 3.12). The API key is read from the
`ANTHROPIC_API_KEY` environment variable — never hard-coded.

```bash
pip install -r agents/guidelines/requirements.txt   # only needed for the LLM path

# Demo on the placeholder patient:
python -m agents.guidelines.demo            # uses the LLM if ANTHROPIC_API_KEY is set
python -m agents.guidelines.demo --offline  # deterministic, no network

# Tests:
python -m pytest agents/guidelines/tests -q         # with pytest
python agents/guidelines/tests/run_all.py           # zero-dependency runner
```

The synthesis model is set in `config.py` (`MODEL = "claude-sonnet-5"`).

### Offline behaviour

With no `ANTHROPIC_API_KEY` (or on any API error) the synthesis step falls back
to a deterministic enrichment built from the matched chart facts, so the whole
pipeline stays runnable and the tests never touch the network. The evidence
fields are identical either way — only the confidence rationale / note wording
differs.

## Tests (what they prove)

- `test_contract.py` — every finding validates against contract (A); exact field
  set; rationale always non-empty.
- `test_fertility_gap.py` — the planted fertility gap is caught as a `gap`, and
  documenting it in the chart flips it to `addressed`.
- `test_grade_integrity.py` — the output `recommendation_grade` (and
  `recommendation` / `evidence_ref`) is byte-for-byte identical to the card,
  **including on the LLM path** with an adversarial stub that tries to smuggle a
  bogus grade — proving the model can never rewrite the evidence.
- `test_triage_and_matcher.py` — triage selection and the generic matcher
  (age bounds, present-data requirement, case-insensitivity, FHIR-ish
  extraction).
